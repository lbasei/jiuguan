# Life Kitchen2

**一个基于不同管理系统调酒师的个人日程协作Agent。**

我们不想把所有人的任务都粗暴归类成几种固定菜单，而是设计了一个“原料萃取系统”。每个待办都会根据任务类型变成一种基础原料，再根据时间、精力、情绪和完成状态产生不同浓度、比例和口感。这样同样是写 PRD，有的人会生成一层清澈的深度茶底，有的人会生成过浓的苦味冷萃，有的人则会因为被消息打断形成断层。最终每个人的一天都会在酒馆中调配成不同的今日特调。

## **1\. 项目概述**

### **1\.1 项目名称**

**Life Kitchen 进化酒馆**

### **1\.2 参赛赛道**

**个人助理与深度个性化赛道**

### **1\.3 一句话介绍**

Life Kitchen 是一个用“调酒”方式管理人生待办的个人助理 Agent。用户的每个待办都会被萃取成一种原料，时间决定比例，任务类型决定风味，完成状态决定口感。Agent 会生成并进化一位专属的“调酒师种种”，帮助用户调配今日计划、优化任务顺序，并生成日度、月度、年度复盘。

## **2\. 产品定位**

它是一个可以定制“谁来管理我、用什么方式管理我”的个人日程调酒 Agent。

用户管理的不是一串待办，而是一杯由任务、时间、精力、情绪和习惯调配而成的“今日特调”。

### **3\.4 今日特调**

一天结束后，系统会根据用户的实际完成情况，生成一杯专属的 **“今日特调”**。

“今日特调”是由当天所有待办事项转化出的原料比例共同调配而成。每个待办都会根据任务类型、预计时间、实际用时、精力消耗、情绪负担和完成状态生成不同原料。最终，这些原料会组合成一杯代表今日生活状态的酒品 / 饮品。

最终展示界面以“酒馆”为场景：用户完成一天任务后，调酒师种种会在吧台中端出今日特调，并生成一张可收藏的特调卡。

#### **今日特调展示内容**

- 酒品名称

- 酒品视觉

- 调酒师种种

- 今日原料比例

- 完成率

- 时间准确度

- 精力消耗

- 情绪浓度

- 任务结构评价

- Agent 复盘点评

- 明日调配建议

#### **评价维度**

#### **示例**

今日待办包括：

- 写 PRD：深度工作，90 分钟，已完成

- 讨论方案：沟通任务，60 分钟，已完成

- 回消息：琐碎事务，20 分钟，延迟

- 整理素材：创作辅助，40 分钟，已完成

- 散步休息：恢复任务，30 分钟，未完成

系统生成：

```Plain Text
今日特调：迷迭香青柠冷萃

配方：
45% 深度茶底
20% 沟通气泡
15% 创意糖浆
10% 琐事小料
10% 恢复奶泡

种种点评：
今天的主茶底很稳，核心任务完成度较高。但恢复奶泡偏少，说明休息被压缩了。明天建议在高强度任务后加入一段固定缓冲，避免整杯特调过浓。
```

#### **产品意义**

“今日特调”承担三个作用：

1. **成果可视化**：把一天的完成情况变成可感知、可收藏的酒品。

2. **复盘轻量化**：用风味、比例和口感替代冷冰冰的数据报表。

3. **个性化进化**：Agent 会根据每日特调结果，持续调整用户的调酒师种种、任务顺序和管理方法。

这版可以直接替换原 PRD 的 **3\.4 今日特调**。

## **5\.1 原料层：待办事项的分类与萃取方法**

### **5\.1\.1 模块定义**

原料层负责把用户输入的每一个待办事项，转化为可以被 Agent 调配的“原料单位”。

在 Life Kitchen 中，待办事项不会被简单归类成固定菜单，而是根据任务属性被萃取成不同类型的原料。每个原料都带有类别、比例、浓度、风味、完成状态和复盘意义。

核心逻辑：

```Plain Text
每个待办事项
↓
根据事件类型分类
↓
根据时间和优先级计算比例
↓

↓
根据完成状态决定口感
↓
进入今日特调配方
```

### **5\.1\.6 原料比例计算**

```Plain Text
单个任务预计时间 ÷ 今日所有任务预计时间 = 该原料初始比例
```

例如：

```Plain Text
写 PRD：90 分钟
讨论方案：60 分钟
回消息：20 分钟
整理素材：40 分钟
休息散步：30 分钟

总预计时间：240 分钟
```

对应比例：

用户可以用自然语言一次性输入当天所有想做、必须做、担心、拖延或临时想到的事情。

系统会由调酒师种种进行整理，将混乱输入拆解成结构化 Todo List，并进一步转化为今日特调的原料。

该功能用于降低用户整理待办的门槛：

```Plain Text

```

### **9\.5 调酒师的配方整理**

调酒师种种会在生成 Todo List 后，继续输出不同风格的配方方案。

同一组待办可以被整理成不同管理风格。

#### **方案 A：迷迭香结构配方**

适合：今天必须完成核心任务。

```Plain Text
写 PRD
↓
整理小精灵设定表
↓
讨论比赛方案
↓
回复老师信息 + 整理报销截图
↓
运动
↓
日度复盘
```

Agent 解释：

```Plain Text
今天的主茶底是 PRD，建议先处理最需要完整时间的任务，再集中处理沟通和琐事。
```

---

#### **方案 B：薄荷恢复配方**

适合：用户疲惫但仍需推进。

```Plain Text
轻整理 10 分钟
↓
写 PRD 45 分钟
↓
休息 / 运动
↓
写 PRD 45 分钟
↓
讨论比赛方案
↓
回复信息和报销截图
↓
日度复盘
```

Agent 解释：

```Plain Text
今天状态偏累，不建议一口气高强度推进。把主任务拆成两段，中间加入恢复奶泡。
```

---

#### **方案 C：姜味启动配方**

适合：用户迟迟不想开始。

```Plain Text
回复老师信息
↓
打开 PRD 文档并写 10 分钟
↓
继续写 PRD
↓
整理小精灵设定表
↓
讨论比赛方案
↓
报销截图
↓
运动和复盘
```

Agent 解释：

```Plain Text
先用一个短任务点火，再进入主任务。重点不是立刻做完，而是先开火。
```

### **9\.9 该模块的产品意义**

自然语言输入让 Life Kitchen 更接近真正的个人助理。

用户不需要先整理好自己的人生，才能使用工具。

用户可以先把混乱倒出来，再由调酒师种种帮忙过滤、萃取、分类和调配。

这也是产品与普通 Todo List 的关键差异：

```Plain Text
普通 Todo List：要求用户自己整理好任务
Life Kitchen：允许用户混乱输入，由 Agent 帮助生成可执行配方
```

项目将 EvoMap 设计为管理经验配方池。Agent 吸收 EvoMap 后，调酒师种种的策略、任务顺序和原料比例会发生变化，进化可视化明确。

---

## **20\.2 技术创新性**

创新点：

1. 将 Todo 数据转化为原料比例

2. 将 Agent 管理风格做成可定制调酒师

3. 用调酒配方表达个人时间结构

4. EvoMap 经验直接影响 Agent 决策

5. 日度复盘变成可视化“今日特调卡”

---

## **20\.4 完成度与表现力**

第一版只展示一条主线：

```Plain Text
定制调酒师
↓
输入待办
↓
生成原料
↓
调配今日特调
↓
吸收 EvoMap 经验
↓
Agent 进化
↓
执行任务
↓
生成复盘
```

视觉表达直接围绕：

```Plain Text
待办 = 原料
时间 = 比例
管理方法 = 调酒师
一天 = 今日特调
复盘 = 配方点评
```
